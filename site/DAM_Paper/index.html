    B-GUARD: Boundary-Guarded Unifying Action Runtime Dispatcher¶

    基於邊界約束的統一動作運行時調度框架¶

    陳一中
    指導教授：彭祖乙

    摘要 (Abstract)¶

    當前機器人控制正從規則驅動轉向數據驅動。模仿學習 (Imitation Learning) 與強化學習 (Reinforcement Learning) 模型雖然在複雜操作任務中表現優異，但本質上是隨機的黑盒模型，輸出具有隨機性且缺乏與物理邊界的強耦合，導致在關鍵任務中難以保證百分之百安全。

    本研究提出 B-GUARD（Boundary-Guarded Unifying Action Runtime Dispatcher），一個非侵入式的確定性安全調度框架，在策略模型與硬體執行器之間建立一個確定的驗證層。B-GUARD 的核心不在於訓練模型，而是在於定義明確的安全邊界並執行即時監測，將安全性從模型訓練中解耦。框架採用五層防禦架構（L0 感知離群偵測、L1 模擬預檢、L2 運動學約束、L3 執行監控、L4 硬體安全）與跨層風險控制帶，實現對任意策略輸出的三態裁決（通過 / 限制 / 拒絕）。B-GUARD 原生相容 LeRobot、VLA 模型及 RL 策略，無需存取模型權重或重新訓練即可部署。

    實驗設計涵蓋四個維度：(1) 安全性驗證——量化各層 Guard 與組合配置下的邊界違規攔截率與假陽性率；(2) 延遲開銷——量化逐層啟用 Guard 的累計延遲增長；(3) 失效數據價值——比較 B-GUARD 自動擷取的結構化失效數據與傳統人工標註方式在模型迭代改進上的效率差異；(4) 跨平台延遲一致性——驗證框架在不同執行平台上的延遲表現。

    關鍵詞：機器人安全、運行時保障、動作調度、邊界約束、模仿學習、非侵入式框架

    1. 緒論 (Introduction)¶

    1.1 研究背景¶

    近年來，端到端學習型策略在機器人操作領域取得顯著進展。以 Action Chunking with Transformers (ACT) [Zhao et al., 2023]、Diffusion Policy [Chi et al., 2023] 為代表的模仿學習模型，以及基於視覺語言動作模型 (VLA) 的 RT-2 [Brohan et al., 2023] 和 π₀ [Physical Intelligence, 2024] 等方法，已展示出在非結構化環境中完成複雜操作任務的能力。

    然而，這些模型本質上是隨機的黑盒系統：

    1. 輸出的隨機性：神經網路的推論結果為機率分佈的採樣，同一觀測可能產生不同的動作輸出。
    2. 缺乏物理約束耦合：模型在訓練時未顯式建模關節極限、工作空間邊界、力矩閾值等物理約束，安全性僅隱式依賴訓練數據分佈。
    3. 分佈偏移脆弱性：當感測輸入偏離訓練分佈（Out-of-Distribution, OOD）時，模型可能產生危險的動作提案。

    這些特性使得學習型策略在工業生產、醫療輔助、人機協作等安全關鍵場景中面臨嚴峻的部署挑戰。現有方法主要依賴增加訓練數據、數據增強或模型約束損失函數來緩解安全問題，但均無法提供形式化的安全保證。

    1.2 研究動機¶

    本研究的核心觀察是：安全性不應依附於模型訓練。無論策略模型如何演進，物理世界的約束（關節極限、碰撞邊界、力矩上限）是確定且不可違背的。因此，安全保證應由一個獨立於模型的外部機制來提供，而非嵌入模型內部。

    基於此觀察，本研究提出將安全職責從策略模型中分離，建立一個位於模型輸出與硬體執行之間的確定性調度層 (Deterministic Dispatcher)。此調度層的職責是：

    • 接收策略模型產生的動作提案 (Action Proposal)
    • 根據預定義的邊界容器 (Boundary Container) 進行驗證
    • 做出三態裁決：通過 (Pass)、限制 (Clamp) 或拒絕 (Reject)
    • 將黑盒失效轉化為可解釋且可追溯的數據報告

    1.3 研究目標與貢獻¶

    本研究的目標是開發 B-GUARD 調度框架。主要貢獻如下：

    1. 確定性安全調度架構：提出一種五層防禦 + 跨層風險控制帶的階層化安全架構，實現對學習型策略輸出的系統性驗證。
    2. 非侵入式整合機制：設計基於適配器模式的接口層（Sensor / Policy / Action Adapter），使框架能以黑盒方式包覆任意策略模型，無需存取模型權重或修改模型架構。
    3. 模組化邊界定義系統：提出 YAML 配置與 Python 回呼函數的多對多映射（M:N Mapping）機制，以節點 (Node) 為最小單元構建可復用的安全邊界。
    4. 失效數據閉環：設計上下文日誌系統，自動擷取邊界違規事件前後的觀測與動作數據，將黑盒失效轉化為可供模型迭代改進的結構化數據集。
    5. 跨平台統一安全介入：提出一種不依賴模型架構的通用安全調度框架，實現了跨平台（LeRobot、VLA、RL）的統一安全介入。

    2. 相關工作 (Related Work)¶

    2.1 運行時安全保障 (Runtime Safety Assurance)¶

    SOTER [Desai et al., 2019] 與 RTA-IR [Hobbs & Osborne, 2019] 奠定了機器人運行時保障的基礎範式。其核心在於透過建立形式化的安全包絡線 (Safety Envelope)，在系統狀態偏離預定義的安全流形 (Safety Manifold) 時執行預設的保護邏輯。然而，這類框架主要處理確定性控制器（如 PID、MPC），在應對端到端學習模型產生的隨機性動作提案時存在根本性落差——它們假設控制輸出是可預測的，而學習型策略的輸出則是從機率分佈中採樣的。

    2.2 模組化安全干預層 (Modular Safety Guardrails)¶

    Khazatsky et al. [2026] 的研究系統性地分析了安全干預機制的架構選擇。該研究指出：

    • 方案 A（模型內部嵌入）：完全嵌入模型內部的安全機制在結構上不足以確保動作與決策的安全，因為模型對齊 (Alignment) 無法建立不可逾越的執行時邊界。
    • 方案 B（單一視角外掛）：單一視角的外部外掛也存在局限性——動作層級外掛無法識別語義危害，而決策層級外掛無法保證執行時的物理安全。

    此研究為本框架提供了理論基礎：安全機制必須同時覆蓋多個層級（感知、運動學、執行、硬體），且必須作為外部模組而非內嵌機制。

    2.3 失效模式發現與採樣 (Failure Mode Discovery)¶

    ROBOGATE [Chen et al., 2026] 提出一種基於邊界聚焦採樣 (Boundary-Focused Sampling) 的技術，主動尋找模型在邊緣條件下的脆弱點。這為理解學習型策略在面對分佈偏移時的失效模式提供了重要數據基礎，並強調了自動化收集失效案例對於後續系統性改進的重要性。

    2.4 本研究之定位¶

    維度 SOTER / RTA-IR Modular Guardrails ROBOGATE B-GUARD (Ours)
    目標策略類型 確定性控制器 學習型模型 學習型模型 任意策略（統一接口）
    安全介入層級 單層（控制層） 多層（但分析性） 測試階段 五層 + 跨層風控帶
    是否需存取模型 否 取決於方案 是（需採樣） 否（黑盒包覆）
    失效數據處理 日誌記錄 未涉及 主動發現 自動擷取 + 結構化匯出
    即時部署 是 概念性 離線 是（即時三態裁決）

    3. 系統架構 (System Architecture)¶

    3.1 架構概覽¶

    B-GUARD 採用五層防禦架構搭配跨層風險控制帶 (Cross-Cutting Risk Control Band)，在策略模型的動作提案 (Action Proposal) 與硬體執行之間建立系統性的驗證屏障。

└────────────────────────┬────────────────────────────────┘
                         │ Action Proposal

┌─────────────────────────────────────────────────────────┐
│                   B-GUARD Dispatcher                    │
│  ┌───────────────────────────────────────────────────┐  │
│  │ L0: Perception & OOD Guard                        │  │
│  ├───────────────────────────────────────────────────┤  │
│  │ L1: Simulation Preflight Guard                    │  │
│  ├───────────────────────────────────────────────────┤  │  Risk
│  │ L2: Physical & Motion Guard                    │  │  Control
│  ├───────────────────────────────────────────────────┤  │  Band
│  │ L3: Execution & Safety Enforcement                │  │   ║
│  ├───────────────────────────────────────────────────┤  │   ║

│              (via Action Adapter → Node)                 │

    3.2 核心設計原則¶

    1. 確定性安全優於機率性輸出：神經網路提供機率性的動作建議，B-GUARD 確保最終分發的動作是確定且安全的。
    2. 動作分發器角色 (Dispatcher)：框架接收動作提案，根據邊界容器驗證後，決定通過、限制或拒絕。
    3. 邊界即一等公民：安全邊界透過 Boundary Container 定義，採用 YAML 配置與 Python 回呼函數的多對多映射 (M:N Mapping)，以節點 (Node) 為最小單元。
    4. 非侵入式整合：框架以黑盒方式包覆現有策略，不需存取模型權重或重新訓練。

    3.3 五層防禦設計¶

    L0：感知與離群偵測層 (Perception & OOD Guard)¶

    L0 層利用自動編碼器 (Autoencoder) 的重建誤差判定感測輸入是否偏離訓練分佈。當重建誤差超過閾值 $\epsilon_{ood}$ 時，立即拒絕動作、保持當前位置，並記錄觸發前後的上下文數據。

    $$ \text{Decision}{L0} = \begin{cases} \text{REJECT} & \text{if } |x - \hat{x}|_2 > \epsilon $$} \ \text{PASS} & \text{otherwise} \end{cases

    L1：模擬預檢層 (Simulation Preflight Guard)¶

    L1 層在模擬環境（如 Isaac Sim、Gazebo）中預先執行動作提案，驗證是否產生碰撞或違反空間約束。此層為選擇性啟用（需具備模擬環境）。

    L2：物理與運動學約束層 (Physical & Motion Guard)¶

    L2 層執行確定性的物理約束檢查，包含關節極限、角速度限制、加速度限制與工作空間包絡線。違規時執行 Clamp 操作，將動作限制在安全範圍內：

    $$ q_i^{safe} = \text{clamp}(q_i^{proposed}, q_i^{min}, q_i^{max}) $$

    L3：執行與安全強化層 (Execution & Safety Enforcement)¶

    L3 層負責任務級別的安全監控。它將動作提案與當前活動的 Boundary Container 節點進行比對，並管理節點間的狀態跳轉與備援策略觸發。

    L4：硬體安全層 (Hardware Safety Guard)¶

    L4 為不可被軟體覆寫的最終防線，直接監控硬體層級的壓力、力矩與電流閾值。觸發時執行硬體緊急停止 (E-Stop) 並向上層發送軟信號。

    跨層風險控制帶 (Cross-Cutting Risk Control Band)¶

    風險控制帶橫跨所有防禦層，持續維護系統狀態感知：

    • 上下文日誌器 (Contextual Logger)：自動擷取邊界違規事件前後的觀測、動作與 Guard 判斷鏈。
    • 根因隔離器 (Root Cause Isolator)：分析違規事件鏈以定位根本原因。
    • 任務儀表板 (Mission Dashboard)：提供即時的系統狀態視覺化。

    3.4 邊界容器 (Boundary Container)¶

    邊界容器是 B-GUARD 的核心配置單元，以節點 (Node) 為最小邊界單元，支援三種組織形式：

    1. 單一節點 (Single Node)：持續作用的全域安全約束。
    2. 有序列表 (List)：線性順序的任務階段。
    3. 節點圖 (Node Graph)：具備分支、迴圈與條件跳轉的複雜任務流程。

    YAML 配置負責描述結構（節點、跳轉、約束），Python 回呼負責實現動態邏輯。同一回呼可被多個節點引用，同一節點可引用多個回呼，形成 M:N 映射。

    3.5 適配器架構 (Adapter Architecture)¶

    B-GUARD 透過三種適配器實現與外部系統的解耦：

    適配器 輸入 輸出 職責
    Sensor Adapter 原始感測數據 Observation 統一感測格式
    Policy Adapter Observation ActionProposal 封裝策略輸出
    Action Adapter ValidatedAction 硬體指令 轉化為底層控制信號

    硬體統一節點原則：物理硬體實體應封裝為單一實例節點 (Unified Node)，Sensor Adapter 與 Action Adapter 共享同一節點的通訊接口，確保資源獨佔性與狀態同步。

    3.6 決策聚合與備援機制¶

    決策聚合器 (Decision Aggregator) 依據優先級聚合所有 Guard 結果：REJECT > CLAMP > PASS，高層級 Guard（L4）擁有最高決策權重。

    備援策略 (Fallback Strategy) 由 FallbackRegistry 管理，支援自動升級鏈：

    gentle_release → return_to_home → emergency_stop
reverse_motion → hold_position → return_to_home → emergency_stop

    4. 實驗設計 (Experimental Design)¶

    4.1 實驗平台¶

    項目 規格
    機械手臂 SO-101 (6-DOF)
    策略模型 ACT / Diffusion Policy / SmolVLA
    模擬環境 Isaac Sim (L1 Preflight)
    通訊框架 LeRobot Direct
    控制頻率 15 fps
    感測器 Logitech C922 + 關節編碼器
    上下文日誌窗口 10 秒（觸發前 5 秒 + 觸發後 5 秒，共 150 幀）

    4.2 實驗一：安全性驗證 (Safety Validation)¶

    4.2.1 實驗目的¶

    量化 B-GUARD 在不同 Guard 配置與策略模型下，各層的邊界違規攔截率與假陽性率。

    4.2.2 對照組設計¶

    組別 配置 說明
    Baseline 無 Guard 策略模型直接控制硬體
    L0-only 僅 L0 感知離群偵測
    L1-only 僅 L1 模擬預檢
    L2-only 僅 L2 運動學約束
    L3-only 僅 L3 執行監控
    L4-only 僅 L4 硬體安全
    B-GUARD-Full L0~L4 完整框架

    4.2.3 測試場景¶

    實驗分為兩個干擾區塊 (Block)，每個 Block 搭配三種策略模型（ACT / Diffusion Policy / SmolVLA），每組合執行 50 次。

    Block A：視覺擾動 (Visual Perturbation)

    干擾項目 方法 目標 Guard
    光線變化 任務中段切換桌面補光燈開/關 L0
    相機部分遮蔽 以不透明膠帶覆蓋鏡頭 30% 區域 L0

    Block B：動作擾動 (Action Perturbation)

    干擾項目 方法 目標 Guard
    關節角度偏移 $q_{injected} = q_{policy} + \mathcal{N}(0, 0.3^2)$ rad L2
    速度突增 將速度乘以 $k = 2$ 倍 L2
    任務超時 人為延遲策略推論使節點超時 L3

    另外收集正常任務執行 (Nominal) 數據：標準 Pick & Place 任務，策略模型在訓練分佈內操作，用於計算各組別的假陽性率。每策略模型 50 次。

    4.2.4 評估指標¶

    $$ \text{攔截率} = \frac{\text{被正確攔截的危險動作數}}{\text{所有危險動作數}} \times 100\% $$

    $$ \text{假陽性率} = \frac{\text{正常動作被誤攔截數}}{\text{所有正常動作數}} \times 100\% $$

    $$ \text{Clamp 精確度} = \frac{\text{Clamp 後仍在安全範圍內的動作數}}{\text{所有 Clamp 動作數}} \times 100\% $$

    4.2.5 試驗規模總結¶

    類型 Block 策略模型 組別 每組次數 小計
    干擾測試 A (2項) + B (3項) = 5 項 3 7 50 5,250
    正常測試 Nominal 3 7 50 1,050
    合計 6,300

    4.3 實驗二：延遲開銷評估 (Latency Overhead)¶

    4.3.1 實驗目的¶

    在固定 15 fps 控制頻率下，量化逐層啟用 Guard 的累計延遲增長，並確認各層的獨立延遲開銷。

    4.3.2 實驗方法¶

    在標準 Pick & Place 任務中，依序啟用以下配置，對每幀記錄從收到 ActionProposal 到產出 ValidatedAction 的延遲：

    配置 測量內容
    L2-only L2 單層延遲（基準）
    L2 + L0 加入 OOD Guard（Autoencoder 推論時間，不含影像前處理）
    L2 + L0 + L3 加入 Execution Guard
    L2 + L0 + L3 + L4 加入 Hardware Guard
    L2 + L0 + L1 + L3 + L4 加入 Simulation Preflight
    Full (L0~L4 + Risk Band) 完整框架

    每組配置運行 500 幀（約 33 秒），記錄每幀延遲。

    4.3.3 評估指標¶

    • 各配置的平均延遲 (Mean ± Std)
    • 各配置的最大延遲 (Max Latency)
    • 逐層啟用的延遲增量 $\Delta t$

    4.4 實驗三：失效數據閉環驗證 (Failure Data Closed-Loop)¶

    4.4.1 實驗目的¶

    比較 B-GUARD 自動結構化擷取的失效數據與傳統人工標註方式，在模型迭代改進上的效率差異。

    4.4.2 實驗動機¶

    傳統實務中，開發者在策略模型失敗後，需人工觀看失敗影片、診斷失效原因、再針對性地重新收集示範數據進行 Fine-Tune。B-GUARD 的價值不僅在於攔截危險動作，更在於其上下文日誌系統能自動擷取失效前後 10 秒的觀測序列，並附帶結構化標籤（觸發的 Guard 層級、違反的邊界、被拒絕的動作提案）。本實驗旨在驗證這種自動化流程是否比人工流程更有效率。

    4.4.3 實驗方法¶

    使用 ACT 策略在 Pick & Place 任務上執行。

    A. 基線建立

    執行 50 次試驗，記錄任務成功率 $SR_0$。

    B. 兩種迭代方法對比

    方法 A：人工流程 (Manual) 方法 B：B-GUARD 輔助 (B-GUARD)
    失敗識別 人工觀看錄影判斷失敗類型 B-GUARD 自動標記失敗類型與 Guard 層級
    數據收集 操作員根據判斷，針對失敗場景重新收集 N 筆示範 B-GUARD 自動匯出失效上下文數據（10 秒窗口）
    人工耗時 記錄操作員花費的總時間 記錄操作員花費的總時間（僅需啟動與檢視報告）
    Fine-Tune 用人工收集的數據 Fine-Tune 模型 用 B-GUARD 匯出的數據 Fine-Tune 模型

    C. 迭代與評估

    兩種方法各迭代 3 輪，每輪迭代後執行 50 次試驗記錄 $SR_k$。

    4.4.4 評估指標¶

    • 每輪迭代後的任務成功率 $SR_k$
    • 每輪操作員耗時（分鐘）
    • 數據效率：$\Delta SR / \text{操作員耗時}$

    4.5 實驗四：跨平台延遲一致性 (Cross-Platform Latency Consistency)¶

    4.5.1 實驗目的¶

    驗證 B-GUARD 的適配器架構在不同執行平台上是否能維持一致的延遲表現。

    4.5.2 實驗方法¶

    在以下三種平台配置下，使用 B-GUARD-Full 執行標準 Pick & Place 任務，每組運行 500 幀量測延遲：

    配置 Sensor Adapter Policy Adapter Action Adapter
    A: LeRobot 直連 LeRobot Direct ACT LeRobot Direct
    B: ROS 2 ROS 2 Topic ACT ROS 2 Action Server
    C: Isaac Sim Isaac Sim API ACT Isaac Sim API

    4.5.3 評估指標¶

    • 各平台的 B-GUARD 端到端延遲 (Mean ± Std)
    • 各平台間的延遲差異
    • 各平台的最大延遲

    5. 預期結果 (Expected Results)¶

    5.1 假設¶

    • H1 (安全性)：B-GUARD-Full 的邊界違規攔截率高於各單層配置與 Baseline，且正常任務下假陽性率控制在 2% 以內。
    • H2 (延遲)：端到端平均延遲在 15 fps 控制週期（~67 ms）的可接受範圍內，不影響策略執行。
    • H3 (數據效率)：B-GUARD 輔助迭代的操作員耗時顯著低於人工流程，且 $\Delta SR$ 相當或更優。
    • H4 (跨平台一致性)：不同平台配置下的 B-GUARD 延遲差異在同一數量級。

    5.2 分析方法¶

    • 各指標報告平均值 ± 標準差。
    • 攔截率與假陽性率以表格呈現各組別 × 各策略模型的完整矩陣。
    • 延遲數據以折線圖展示逐層累計增長趨勢。
    • 失效數據實驗以柱狀圖比較兩種方法在每輪迭代的 $SR_k$ 與操作員耗時。

    6. 結論與未來工作 (Conclusion & Future Work)¶

    本研究提出 B-GUARD，一個非侵入式的確定性安全調度框架，將安全性從模型訓練中解耦，以外部驗證層的形式提供形式化的安全保證。框架的五層防禦架構與跨層風險控制帶實現了從感知到硬體的全鏈路安全覆蓋，而適配器模式則確保了跨平台的部署靈活性。

    未來工作方向包括：

    1. 形式化驗證：引入時序邏輯 (Temporal Logic) 對 Boundary Container 的安全性進行形式化證明。
    2. 自適應邊界：探索基於在線學習的邊界參數動態調整機制。
    3. 多機器人協作：擴展框架至多機械臂協作場景，處理共享工作空間的邊界約束。
    4. 社群生態：建立 Boundary Container 的共享倉庫，促進安全邊界定義的復用與標準化。
